# 複数の引用符が正しくレンダリングされない問題

## 概要

現状の実装では、引用符 (`>`) が複数個連なってネストされている場合（例: `>> hoge`）や、連続した行に引用符が複数ある場合（例: `> hoge\n> fuga`）に、2つ目以降の引用符が正しくレンダリングされない。

## 原因

`block_quote` ノードをキャプチャし、その子ノードを走査して `>` マーカーを探す現在のロジックでは、全てのケースを網羅できていない。特に、連続した行にあるマーカーの検出に問題がある。

## 対策

より具体的で堅牢な方法として、Tree-sitterのクエリを直接 `block_quote_marker` ノードにマッチするように変更する。これにより、`block_quote` のネスト構造や連続した行に関わらず、全ての `>` マーカーを個別にキャプチャしてレンダリング処理を適用できる。

### Tree-sitter クエリの変更

#### 変更前
```lua
(block_quote) @quote
```

#### 変更後
```lua
(block_quote_marker) @quote
```

### 元のMarkdown
```markdown
> quote 1
>> quote 2
>>> quote 3
>
> quote 4
```

### 期待される表示
（`config.icons.quote` が `┃` の場合）
```
┃ quote 1
┃┃ quote 2
┃┃┃ quote 3
┃
┃ quote 4
```
