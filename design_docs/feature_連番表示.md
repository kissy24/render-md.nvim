# 機能追加: 連番（Ordered List）の表示対応

## 現状
現在の実装では、`list_item` 内のマーカーが `list_marker`（`-`, `*`, `+`）であるかに関わらず、マーカーを `config.icons.bullet` で上書きしてしまっているため、連番（`1.`, `2.` など）も箇条書きのドットになってしまう。

## 目的
連番リスト（Ordered List）の場合は、元の数字を維持しつつ、適切なハイライトを適用するか、あるいは連番として認識してそのまま表示するように修正する。

## 仕様
1. `list_item` を処理する際、そのマーカーが箇条書き記号（`-`, `*`, `+`）なのか、連番（`1.`, `2.` など）なのかを判別する。
2. 箇条書き記号の場合は、従来通り `config.icons.bullet` を表示する。
3. 連番の場合は、以下のいずれかの対応を行う：
   - そのまま表示する（`conceal` しない）。
   - あるいは、数字部分を維持したままハイライトのみ適用する。
   - Tree-sitter の `list_marker` ノードのタイプを確認し、数字を含む場合は箇条書きアイコンへの置換を行わないようにする。

## 実装方針
`lua/render-md/core.lua` の `item` キャプチャ処理において、ノードのタイプを確認するロジックを修正する。
`list_marker` ノードが数字で始まる（例：`1.`）場合は、`virt_text` による上書きをスキップするか、数字を活かす形にする。
具体的には、マーカーが `list_marker` かつテキストが数字で終わる（`.` または `)` の前が数字）かどうかを判定する。
