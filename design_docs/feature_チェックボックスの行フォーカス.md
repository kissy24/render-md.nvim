# 機能追加: チェックボックス行のフォーカス時生テキスト表示

## 目的
ノーマルモードにおいて、チェックボックス（タスクリスト）が含まれる行にカーソルがある場合、その行だけレンダリングを解除して生のMarkdownテキスト（`[ ]` や `[x]`）を表示するようにし、編集や状態確認を容易にする。

## 仕様
1. ノーマルモードにおいて、カーソルが移動した際（`CursorMoved`）に、現在の行がタスクリストアイテムであるか判定する。
2. 現在の行がタスクリストである場合、その行のレンダリング（extmarks）を一時的に解除するか、あるいは `concealcursor` の設定を調整して、カーソル行の conceal を無効化する。
3. ネイティブの `concealcursor` 設定を利用するのが最も効率的である。現在は `nc` (Normal, Command-line) に設定されているため、カーソル行でも conceal が有効になっている。これを `n` を除外するように調整するか、あるいは特定の要素（チェックボックス）のみを対象とする場合は `CursorMoved` での制御が必要になる。
4. 今回の要望は「チェックボックスに関しては」とのことだが、Neovimの `concealcursor` を `c` (あるいは `nv` から `n` を抜く等) に設定することで、カーソル行の conceal を解除できる。

## 実装方針
- `lua/render-md/init.lua` において、`concealcursor` の設定を `"nc"` から `"c"`（あるいは空文字列、もしくは要望に合わせて調整）に変更する。
- もし「チェックボックスのみ」に限定し、かつ「他のレンダリング（見出しのアイコン等）は維持したい」場合は、`CursorMoved` イベントでカーソル行の extmark を動的に削除・再描画する必要があるが、UXとしては `concealcursor` の調整が一般的かつ軽量である。
- 今回は、チェックボックスの編集を容易にすることが主目的であるため、`concealcursor` を空文字列 `""` に設定し、カーソル行のすべての conceal（チェックボックス、太字記号、リンク等）を解除する方向で実装する。これにより、チェックボックス行にカーソルが来た時に `[ ]` が見えるようになる。
